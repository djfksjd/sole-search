# sole-search — 소상공인 지원사업 전수조사 스킬 설계 스펙

- **작성일**: 2026-07-20
- **상태**: 사용자 승인 완료 (구현 계획 작성 전)
- **형제 스킬**: [`djfksjd/ir-search`](https://github.com/djfksjd/ir-search) — 구조·DNA 계승
- **장기 통합처**: KKACHI 워크스페이스 Phase 2B (소상공인·정책자금 확장)의 백엔드 스킬 후보

## 1. 목적과 포지셔닝

이미 가게·사업을 운영 중인 **소상공인**(치킨집, 카페, 미용실, 온라인셀러 등)이 자신에게 해당되는
정부·지자체 지원사업을 전수조사받고, "즉시 신청 가능한 것"부터 마감순으로 받아보는 스킬.

- **ir-search와의 경계**: ir-search는 "아이템/프로젝트를 가진 (예비)창업자·스타트업"용,
  sole-search는 "운영 중인 소상공인"용. 트리거 문구를 상호 배타적으로 설계한다.
  - sole-search 트리거: "우리 가게 지원사업", "소상공인 지원금", "정책자금 알아봐줘", "소상공인24", "가게 지원 뭐 있어"
  - ir-search에 남는 트리거: "우리 아이템", "창업지원", "K-Startup", "입주공간", "R&D 과제"
- **사용자**: 소상공인 본인이 직접 사용하는 것을 기본으로 한다. 인터뷰·보고서는 쉬운 말로,
  전문용어에는 괄호 해설을 붙인다. 동시에 프로필·산출물은 정형 포맷으로 저장해
  KKACHI 제품이 나중에 그대로 파싱할 수 있게 한다.

## 2. 지원 범위 (전 범위 + 성격 3분류)

| 분류 | 내용 | 소스 중심 |
|---|---|---|
| 💰 받는 돈 | 보조금·바우처 (스마트상점, 온라인판로, 마케팅, 시설개선 등) | 소상공인24, 기업마당 |
| 🏦 빌리는 돈 | 정책자금 대출·보증 (소진공 직접대출, 지역신보 보증) — "갚아야 하는 돈" 명시, 한도·금리·보증기관 표기 | 소진공(ols.semas.or.kr) |
| 🎓 배우고 돕기 | 교육·컨설팅·멘토링·재기지원(희망리턴패키지) | 소상공인24, 기업마당 |

지자체 사업은 기업마당(bizinfo)의 지자체 공고 통합분으로 커버한다.

## 3. 워크플로

### 0단계 — 대화형 인터뷰 → `sole-profile.md`

소상공인은 참조할 프로젝트 문서가 없으므로 **최초 1회 대화형 인터뷰가 입구**다.
쉬운 말로, 한 번에(여러 턴으로 쪼개지 않음) 묻는다:

1. 무슨 장사/사업을 하시나요? (예: 치킨집, 네일샵, 스마트스토어) → 업태·종목 추정
2. 가게(사업장)는 어느 동네인가요? (시·군·구까지) → 지자체 사업 판정
3. 사업자 등록은 하셨나요? 했다면 언제부터? → 업력 축
4. 혼자 하시나요, 직원이 있나요? 몇 명? → 상시근로자 수 (소상공인 법적 판정: 업종별 5인/10인 미만)
5. 대표님 나이대와 성별은? → 청년·여성 특화 사업
6. 지금 제일 필요한 건? (복수 선택): 안 갚는 돈 / 빌리는 돈 / 가게 시설 / 온라인 판매 / 홍보 / 배우기

프로필은 YAML 프론트매터를 가진 `sole-profile.md`로 현재 작업 폴더에 저장한다.
`마지막 조사` 필드로 diff 모드가 직전 보고서를 찾는다. 대화·폴더에서 이미 파악된 항목은 다시 묻지 않는다.
재사용 시 요약을 보여주고 "바뀐 것 있나요?" 한 번만 확인한다.

```markdown
---
type: sole-search-profile
업종: <업태/종목>
지역: <시/군/구>
사업자: <미등록 | YYYY-MM 등록 (N년차)>
상시근로자: <N명>
대표: <나이대 / 성별>
필요: [보조금, 정책자금, ...]
마지막조사: <보고서 폴더 경로> (YYYY-MM-DD)
---
# sole-search 프로필
<한 줄 요약>
```

### 1단계 — 전수 수집

동봉 크롤러 (모집중 공고 전수 수집, jsonl 출력):

| 스크립트 | 소스 | 비고 |
|---|---|---|
| `sbiz_crawl.py` | 소상공인24 (sbiz24.kr) | **신규 개발. 구현 전 기술 스파이크 필수** (렌더링 방식·API 존재 여부 확인) |
| `sources_crawl.py` | 기업마당(bizinfo) 소상공인 분야 + 지자체 | ir-search의 검증된 코드 이식·개조 |
| (동일 스크립트 서브커맨드 또는 references 수동절차) | 소진공 정책자금 공고 | 크롤링 난이도 높으면 `references/sources.md`에 수동확인 절차로 폴백 |

### 2단계 — 판정 (A/B/C)

ir-search의 3단계 판정 유지: **A 즉시 지원 가능 / B 요건 충족 시 / C 변형하면 가능**.
후보는 반드시 상세공고 원문에서 검증하며, 검증 축은 소상공인용으로 교체:

- 업종 제한 (제외 업종: 유흥·사행 등)
- 업력 (창업 N년 이내/이상)
- 상시근로자 수 상한
- 소재지 (지자체 사업)
- 매출 상한 (간이과세·연매출 기준)
- 기수혜 제외 (동일사업 재지원 불가 등)

**추정 금지 원칙**: 공고 원문에 없는 것은 '불명'으로 표기하고 접수기관 유선확인을 권고한다.
유선확인용 전화번호를 보고서에 표기한다.

### 3단계 — 보고서

최상위 구조는 성격 3분류(💰/🏦/🎓), 각 분류 안에서 A/B/C, 마감 D-day 오름차순 정렬.
사장님이 읽는 문서 톤 — 관공서 용어에 괄호 해설. 🏦 항목에는 "갚아야 하는 돈" 문구와
한도·금리·보증기관을 반드시 표기한다.

### diff 모드 (재조사)

`diff_surveys.py`를 ir-search에서 이식. 프로필의 `마지막조사` 폴더가 있으면 증분 조사:
같은 소스 구성으로 재크롤 → 신규분만 검토·검증 → 보고서는
`신규 / 마감 변경 / 소멸된 A그룹 / 승계 요약` 구조. WARNING 소스는 "미갱신"으로 명시.

## 7. 테스트

- **크롤러 스모크**: 실제 사이트 대상 실행 — 건수 > 0, 필수 필드(제목·기관·마감·URL) 채움률 검증
- **트리거 evals** (`evals/evals.json`): "우리 치킨집에 맞는 지원금 찾아줘" 류 시나리오 +
  ir-search로 가야 할 문구가 sole-search를 트리거하지 않는 네거티브 케이스
- **엔드투엔드**: 가상 프로필 1개(예: 서울 마포구 3년차 카페, 직원 1명)로
  전체 파이프라인 1회 실행, 보고서 산출 확인
